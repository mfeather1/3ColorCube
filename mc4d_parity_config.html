Magic Cube 4D
Edge Parity Configuration

This hypercube has one pair of swapped edge cubies on both the outer and middle layers so each of the two orbits is a "parity" configuration. This can be solved using the same method as the 4x4 Cube & the Void Cube. The Parity macro solves this config, it works by quarter-twisting the outer layer of the horizontal slice (QtrSlice macro) and then using Sequence 4C-E3 twice to flip all four edges in the slice. Sequence 4C-E1m swaps the center cubies and the rest is cleanup using the 8-color edge sequences (139 moves total). The setup for the Parity macro has the hypercube rotated so that the misplaced facelets are in the vertical slice as shown.
